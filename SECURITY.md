# Security Policy

## Supported Versions

Only the latest release is supported with security fixes. Patches are released as soon as possible after a vulnerability is confirmed.

| Version | Supported |
| ------- | --------- |
| latest  | yes       |
| older   | no        |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report vulnerabilities privately using one of the following methods:

- **GitHub private security advisory** (preferred):
  [https://github.com/JamesWoolfenden/ghat/security/advisories/new](https://github.com/JamesWoolfenden/ghat/security/advisories/new)
- **Email**: [james.woolfenden@gmail.com](mailto:james.woolfenden@gmail.com)

### What to include

To help reproduce and assess the issue, please provide:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Affected versions
- Any suggested remediation if known

### Disclosure timeline

| Milestone | Target |
| --------- | ------ |
| Acknowledgement of report | within 2 business days |
| Confirmation of vulnerability | within 7 days |
| Fix released | within 90 days of confirmation |
| Public disclosure | after fix is released, coordinated with reporter |

If a fix cannot be delivered within 90 days, the reporter will be notified with an updated timeline before public disclosure proceeds.

### Coordinated disclosure

This project follows [coordinated vulnerability disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure). Reporters who follow this policy will be credited in the release notes unless they prefer to remain anonymous.
